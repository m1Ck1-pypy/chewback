# FAQ (Часто задаваемые вопросы) - Chewback

## Общие вопросы

### Что такое Chewback?
Chewback - это современный десктопный мессенджер с акцентом на безопасность, приватность и производительность. Приложение построено на стеке Tauri + SolidJS + Rust и предлагает безопасный способ обмена сообщениями.

### Чем Chewback отличается от других мессенджеров?
- **Безопасность:** End-to-end шифрование, безопасное хранение токенов в системном keyring
- **Производительность:** Нативная десктопная версия без Electron overhead
- **Приватность:** Открытый исходный код, минимальная коллекция данных
- **Архитектура:** Современный стек технологий (Rust, SolidJS, Tauri)

### На каких платформах работает Chewback?
- **Windows:** 10 и выше (x64, ARM64)
- **macOS:** 11.0 (Big Sur) и выше (Intel, Apple Silicon)
- **Linux:** Современные дистрибутивы с поддержкой Wayland/X11

### Сколько стоит Chewback?
Chewback полностью бесплатен и с открытым исходным кодом. В будущем могут появиться премиум-функции для корпоративных пользователей, но основная функциональность останется бесплатной.

## Технические вопросы

### Какие технологии используются в проекте?
- **Фронтенд:** SolidJS, TypeScript, Tailwind CSS
- **Десктопная оболочка:** Tauri 2.0, Rust
- **Сервер:** Rust, Axum, PostgreSQL, SQLx
- **Безопасность:** JWT, Argon2, системный keyring

### Почему выбрали Tauri вместо Electron?
Tauri предлагает несколько преимуществ:
- **Размер:** ~5MB против ~100MB у Electron
- **Производительность:** Нативный Rust бэкенд
- **Безопасность:** Встроенная система разрешений
- **Ресурсы:** Меньшее потребление памяти и CPU

### Как обеспечивается безопасность данных?
1. **Пароли:** Хеширование Argon2 с уникальной солью
2. **Токены:** JWT с разделением access/refresh токенов
3. **Хранение:** Refresh токены в системном keyring
4. **Транспорт:** HTTPS, подготовленные SQL запросы
5. **Сообщения:** End-to-end шифрование (в разработке)

### Как работает аутентификация?
1. Пользователь вводит логин и пароль
2. Сервер проверяет пароль (Argon2) и создает сессию
3. Возвращаются два токена:
   - Access token (15 минут) - для API запросов
   - Refresh token (7 дней) - для получения новых access токенов
4. Refresh token сохраняется в системном keyring
5. Access token автоматически обновляется перед истечением

## Установка и настройка

### Как установить Chewback?
1. **Скачайте установщик** с официального сайта
2. **Запустите установку** и следуйте инструкциям
3. **Зарегистрируйтесь** или войдите в существующий аккаунт

### Требования к системе
- **Оперативная память:** 512MB минимум, 2GB рекомендуется
- **Дисковое пространство:** 50MB для приложения + место для сообщений
- **Интернет:** Стабильное подключение для синхронизации

### Как настроить собственный сервер?
1. Установите Rust, Node.js/Bun, PostgreSQL
2. Клонируйте репозиторий: `git clone https://github.com/your-org/chewback`
3. Настройте базу данных: `docker-compose up -d && sqlx migrate run`
4. Запустите сервер: `cd server && cargo run`
5. Соберите клиент: `bun run tauri build`

### Поддерживается ли офлайн режим?
Да, Chewback поддерживает ограниченный офлайн режим:
- Просмотр истории сообщений
- Написание сообщений (сохраняются в очередь)
- Автоматическая отправка при восстановлении соединения

## Использование

### Как начать общение?
1. Зарегистрируйте аккаунт
2. Найдите пользователей через поиск
3. Начните новый чат
4. Отправляйте сообщения в реальном времени

### Какие типы сообщений поддерживаются?
- **Текстовые сообщения** с форматированием Markdown
- **Изображения** (JPEG, PNG, GIF, WebP)
- **Файлы** любого типа до 100MB
- **Голосовые сообщения** (в разработке)
- **Видеозвонки** (планируется)

### Как работает поиск сообщений?
- **Полнотекстовый поиск** по истории сообщений
- **Фильтрация** по дате, отправителю, типу контента
- **Быстрый поиск** контактов по имени

### Есть ли групповые чаты?
Групповые чаты находятся в разработке и будут включать:
- Создание групп до 100 участников
- Администрирование (роли, разрешения)
- Общие файлы и медиа
- Настройки уведомлений

## Безопасность и приватность

### Какие данные собирает Chewback?
Минимальный набор данных для работы:
- **Аккаунт:** Логин, хеш пароля
- **Сообщения:** Зашифрованное содержимое
- **Метаданные:** Время отправки, отправитель, получатель
- **Технические:** Версия приложения, ошибки (анонимно)

### Где хранятся мои сообщения?
- **Локально:** На вашем устройстве в зашифрованном виде
- **На сервере:** В базе данных с end-to-end шифрованием
- **Резервные копии:** Опционально, с дополнительным шифрованием

### Могу ли я экспортировать свои данные?
Да, Chewback предоставляет инструменты для экспорта:
- **Сообщения:** В форматах JSON, HTML, TXT
- **Медиа:** Оригинальные файлы
- **Настройки:** Конфигурация приложения
- **Полный бэкап:** Все данные в архиве

### Как работает end-to-end шифрование?
1. Каждый пользователь имеет пару ключей (публичный/приватный)
2. Сообщения шифруются публичным ключом получателя
3. Только получатель может расшифровать сообщение своим приватным ключом
4. Ключи хранятся локально и никогда не передаются на сервер

## Проблемы и устранение неполадок

### Приложение не запускается
1. **Проверьте совместимость:** Убедитесь, что ваша ОС поддерживается
2. **Переустановите:** Удалите и установите заново
3. **Проверьте антивирус:** Некоторые антивирусы могут блокировать приложение
4. **Посмотрите логи:** `%APPDATA%/chewback/logs` (Windows) или `~/.config/chewback/logs` (Linux/macOS)

### Не могу войти в аккаунт
1. **Проверьте подключение:** Убедитесь, что есть доступ к интернету
2. **Сбросьте пароль:** Используйте функцию восстановления пароля
3. **Очистите кэш:** Удалите файлы в директории данных приложения
4. **Проверьте сервер:** Убедитесь, что сервер доступен

### Сообщения не отправляются
1. **Проверьте соединение:** Убедитесь, что есть интернет
2. **Перезапустите приложение:** Иногда помогает перезагрузка
3. **Проверьте получателя:** Убедитесь, что пользователь существует
4. **Посмотрите очередь:** Сообщения могут быть в очереди отправки

### Медленная работа приложения
1. **Закройте другие приложения:** Освободите память и CPU
2. **Очистите кэш:** Удалите временные файлы
3. **Обновите приложение:** Установите последнюю версию
4. **Проверьте диск:** Убедитесь, что есть свободное место
