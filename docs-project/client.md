# Клиентская часть проекта Chewback

## Обзор

Клиентская часть Chewback - это десктопное приложение, построенное на стеке Tauri + SolidJS + TypeScript. Приложение предоставляет пользовательский интерфейс для мессенджера с акцентом на безопасность и производительность.

## Архитектура

### Технологический стек

- **Tauri 2.0**: Десктопная оболочка с нативным доступом к системе
- **SolidJS**: Реактивный UI фреймворк
- **TypeScript**: Статическая типизация
- **Tailwind CSS**: Утилитарный CSS фреймворк
- **@solidjs/router**: Клиентская маршрутизация
- **@tauri-apps/api**: API для взаимодействия с Tauri

## Ключевые компоненты

### 1. Система аутентификации

#### AuthContext

Центральный контекст для управления состоянием авторизации:

```typescript
interface AuthContextType {
  user: () => User | null;
  isLoading: () => boolean;
  checkAuth: () => Promise<void>;
  logout: () => Promise<void>;
  isAuthenticated: () => boolean;
  refreshToken: () => Promise<boolean>;
  restoreSession: () => Promise<boolean>;
}
```

**Особенности:**

- Автоматическая проверка авторизации при загрузке
- Восстановление сессии из локального хранилища
- Периодическое обновление токенов (каждые 30 минут)
- Очистка интервалов при размонтировании

#### API клиент аутентификации

Класс `AuthApi` наследуется от `BaseApiClient` и предоставляет:

- `register()`: Регистрация нового пользователя
- `login()`: Вход в систему
- `refreshToken()`: Обновление access токена
- `logout()`: Выход из системы
- `getMe()`: Получение данных текущего пользователя

### 2. Базовый API клиент

Класс `BaseApiClient` реализует:

- Универсальный метод `request()` для HTTP запросов
- Обработку ошибок с преобразованием в `ApiError`
- Поддержку всех HTTP методов (GET, POST, PUT, DELETE, PATCH)
- Работу с query параметрами и заголовками

**Особенности реализации:**

- Все запросы проходят через Tauri команду `api_request`
- Автоматическое добавление Content-Type заголовка
- Парсинг ошибок в формате "HTTP {status}: {message}"

### 3. Tauri команды

#### Основные команды:

- `api_request()`: Отправка HTTP запросов через Rust клиент
- `save_refresh_token()`: Сохранение refresh токена в keyring
- `get_refresh_token()`: Получение refresh токена из keyring
- `clear_refresh_token()`: Удаление refresh токена
- `save_user_data()`: Сохранение данных пользователя в Store
- `get_user_data()`: Получение данных пользователя из Store
- `clear_auth_data()`: Очистка всех данных авторизации

#### Безопасное хранение:

- **Refresh токены**: Хранятся в системном keyring
- **Данные пользователя**: Хранятся в Tauri Store (store.json)
- **Cookies**: Сохраняются автоматически через reqwest клиент

### 4. Маршрутизация

Приложение использует файловую маршрутизацию SolidJS:

```typescript
function App() {
  return (
    <AuthProvider>
      <Router>
        <Route path="/login" component={Auth} />
        <Route path="/" component={() => <ProtectedRoute children={<Home />} />} />
      </Router>
    </AuthProvider>
  );
}
```

#### ProtectedRoute

Компонент-обертка для защиты маршрутов:

- Проверяет авторизацию пользователя
- Показывает loading состояние
- Перенаправляет на /login если пользователь не авторизован

### 5. Страницы

#### AuthPage

Страница аутентификации с двумя режимами:

- **Логин**: Вход существующего пользователя
- **Регистрация**: Создание нового аккаунта

**Особенности:**

- Валидация полей ввода
- Обработка ошибок сервера
- Переключение между режимами

#### HomePage

Главная страница приложения (в разработке):

- Будет содержать интерфейс чата
- Список контактов
- Окно переписки

## Поток данных

### Авторизация пользователя

1. **Вход в систему:**

   ```
   Пользователь → AuthPage → AuthApi.login() → Tauri команда → Сервер
         ↑                                          ↓
         └────── Сохранение токенов ←── Ответ ←─────┘
   ```

2. **Автоматическое восстановление:**

   ```
   Загрузка приложения → AuthContext → Проверка Store → Восстановление сессии
            ↓                    ↓
       Показ loading    Обновление состояния
   ```

3. **Обновление токенов:**
   ```
   Интервал (30 мин) → refreshToken() → Новые токены → Обновление Store
   ```

### HTTP запросы

```
Компонент → API клиент → Tauri команда → Rust HTTP клиент → Сервер
     ↑           ↑            ↑                ↑              ↓
     └── Ответ ←─┴── Ответ ←──┴── Ответ ←──────┴────── Ответ ←─┘
```

## Безопасность

### Хранение данных

1. **Refresh токены:**
   - Хранятся в системном keyring
   - Используют нативные API безопасности (Windows Credential Manager, macOS Keychain, Linux Secret Service)
   - Изолированы от файловой системы приложения

2. **Данные пользователя:**
   - Хранятся в зашифрованном Store Tauri
   - Локально на устройстве пользователя
   - Очищаются при выходе из системы

3. **Cookies:**
   - Управляются Rust HTTP клиентом
   - Сохраняются между сессиями
   - Используются для поддержания сессии

### Защита от атак

- **CSRF**: Использование SameSite cookies
- **XSS**: Content Security Policy в Tauri
- **Утечка токенов**: Короткое время жизни access токенов
- **Брутфорс**: Rate limiting на сервере
