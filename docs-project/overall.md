# Общий план разработки мессенджера (Tauri + Solid + Axum)

Этот документ объединяет этапы разработки серверной и клиентской частей, описывая их взаимодействие и общую последовательность работ.

## Текущее состояние проекта

### ✅ Завершено (Этапы 1-2)

1. **Серверная часть (Rust + Axum + PostgreSQL)**
   - Настроена базовая структура проекта с модулями
   - Реализована система миграций базы данных
   - Созданы модели пользователей и сессий
   - Реализована полная система аутентификации:
     - Регистрация пользователей с валидацией
     - Логин с проверкой пароля (Argon2)
     - JWT токены (access + refresh)
     - Механизм обновления токенов
     - Выход из системы с очисткой сессий
   - Настроена система rate limiting
   - Реализованы OpenAPI спецификации
   - Настроен Swagger UI для документации API

2. **Клиентская часть (Tauri + SolidJS + TypeScript)**
   - Настроена базовая структура Tauri приложения
   - Реализована система маршрутизации (@solidjs/router)
   - Созданы страницы аутентификации (логин/регистрация)
   - Реализован глобальный контекст авторизации
   - Создан API клиент с поддержкой Tauri команд
   - Реализовано безопасное хранение токенов:
     - Refresh токены в системном keyring
     - Данные пользователя в Tauri Store
   - Настроена автоматическая проверка и обновление токенов
   - Сгенерированы TypeScript типы из OpenAPI спецификации

3. **Инфраструктура**
   - Настроен Docker Compose для PostgreSQL
   - Созданы скрипты для разработки (Justfile)
   - Настроена сборка для production

### 🚧 В процессе разработки

- **Этап 3**: Интеграция WebSocket для реального времени
- **Этап 3**: Сохранение сообщений в БД
- **Этап 3**: История сообщений и список пользователей

### 📋 Планируется

- **Этап 4**: Поиск пользователей, индикаторы активности

## Архитектура проекта

### Компоненты системы

1. **Клиентское приложение (Desktop)**
   - Фронтенд: SolidJS + TypeScript + Tailwind CSS
   - Десктопная оболочка: Tauri 2.0
   - Хранение данных: Tauri Store + системный keyring

2. **Серверная часть (Backend)**
   - Веб-фреймворк: Axum (Rust)
   - База данных: PostgreSQL
   - ORM: SQLx (async)
   - Аутентификация: JWT + Argon2

3. **Коммуникация**
   - HTTP API: RESTful endpoints для управления данными
   - Tauri команды: Безопасный доступ к системным API
   - WebSocket: Реальное время (в разработке)

### Поток данных

```
Пользователь → Tauri App → HTTP API → Сервер → PostgreSQL
       ↑          ↓           ↑         ↓
       └──────────┴──── Tauri ┴─────────┘
           команды         Store
```

## Ключевые особенности реализации

### Безопасность

- **Пароли**: Хеширование с Argon2
- **Токены**: JWT с коротким временем жизни access токенов
- **Хранение**: Refresh токены в системном keyring
- **Защита**: Rate limiting для аутентификационных endpoints

### Надежность

- **Транзакции**: Все операции с БД в транзакциях
- **Обработка ошибок**: Единая система ошибок на сервере
- **Валидация**: Входные данные валидируются на всех уровнях

### Разработка

- **Типобезопасность**: TypeScript + Rust + SQLx
- **Документация**: OpenAPI + Swagger UI
- **Миграции**: Автоматическое управление схемой БД
- **Тестирование**: Модульные и интеграционные тесты
