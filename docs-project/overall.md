# Общий план разработки мессенджера (Tauri + Solid + Axum)

Этот документ объединяет этапы разработки серверной и клиентской частей, описывая их взаимодействие и общую последовательность работ.

## Этап 0: Предварительная подготовка

- Определить требования: функционал (регистрация, личные сообщения, история), нефункциональные требования (безопасность, производительность).
- Выбрать стек: подтверждено.
- Создать репозиторий на GitHub/GitLab.

## Этап 1: Серверная часть — базовый каркас

- **1.1.** Инициализация Rust проекта, добавление зависимостей.
- **1.2.** Настройка PostgreSQL в Docker, миграции.
- **1.3.** Реализация моделей и репозиториев с поддержкой транзакций.
- **1.4.** Написание модульных тестов для репозиториев.
- **1.5.** Создание HTTP-эндпоинтов регистрации/логина с JWT.
- **1.6.** Интеграционные тесты для аутентификации.
- **1.7.** WebSocket хендлер с broadcast-каналом (без сохранения в БД пока).
- **1.8.** Тестирование WebSocket через тестовые клиенты.

**Результат**: работающий сервер с базовой аутентификацией и возможностью отправки сообщений через WebSocket (без сохранения).

## Этап 2: Клиентская часть — база и интерфейс

- **2.1.** Инициализация Tauri + Solid + Tailwind + Kobalte.
- **2.2.** Настройка роутинга (@solidjs/router).
- **2.3.** Создание страниц логина и регистрации, подключение к серверу.
- **2.4.** Создание глобального состояния (auth, chat).
- **2.5.** Вёрстка основного интерфейса: список контактов, окно чата.
- **2.6.** Компонент сообщения (ChatBubble) с использованием Tailwind.
- **2.7.** Интеграция WebSocket: подключение, приём/отправка сообщений.
- **2.8.** Модульные тесты компонентов.

**Результат**: клиент может авторизоваться, видеть список контактов (заглушки), обмениваться сообщениями через WebSocket в реальном времени.

## Этап 3: Интеграция клиента с полным функционалом сервера

- **3.1.** На сервере: добавить сохранение сообщений в БД (в WebSocket хендлере).
- **3.2.** На сервере: реализовать эндпоинты для получения истории сообщений и списка пользователей.
- **3.3.** На клиенте: загружать историю при открытии чата.
- **3.4.** На клиенте: получать список пользователей с сервера для отображения контактов.
- **3.5.** На клиенте: обрабатывать офлайн-сообщения (если получатель не в сети, сообщение доставится при следующем подключении через запрос истории).

**Результат**: полноценный мессенджер с хранением истории.

## Этап 4: Доработки и тестирование

- **4.1.** Реализовать поиск пользователей.
- **4.2.** Добавить индикаторы "печатает", прочитано (опционально).
- **4.3.** Написать недостающие тесты (интеграционные и E2E).
- **4.4.** Провести ручное тестирование, отловить баги.
- **4.5.** Оптимизировать производительность (пагинация сообщений, виртуализация списка).
- **4.6.** Сборка и подготовка установщиков (Tauri build).

## Этап 5: Расширение (будущие версии)

- Внедрение сквозного шифрования.
- Добавление групповых чатов.
- Поддержка отправки файлов.
- Реализация голосовых/видеозвонков (WebRTC).

## Взаимодействие компонентов

- **Клиент ↔ Сервер (HTTP)**: регистрация, логин, получение списков, истории.
- **Клиент ↔ Сервер (WebSocket)**: отправка/получение сообщений в реальном времени.
- **Rust-часть Tauri**: может использоваться для криптографии, работы с файловой системой, системных уведомлений.
